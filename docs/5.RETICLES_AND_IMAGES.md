## Entity Fields

### Sight

| Field                     | Type      | Semantics                                                                                                                                                     |
| ------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `reticleImage`            | `String?` | Grid type identifier (e.g. `"IHR"`, `"DDR-2"`, `"MIL-XT"`). Taken from `collection.json`, stored in OB. Format is an identifier string, not a path or base64. |
| `calibratedMagnification` | `double`  | Magnification at which the reticle is calibrated (relevant for SFP sights). `-1.0` = not specified.                                                           |
| `image`                   | `String?` | Sight image (photo, art). Format TBD. Currently `null` everywhere, stable in UI.                                                                              |

### Weapon

| Field   | Type      | Semantics                                                                                                                                |
| ------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `image` | `String?` | ID SVG file of a weapon (without extension). For example `"default"` → `assets/svg/weapon/default.svg`. `null` — fallback and `default`. |

### Ammo

| Field   | Type      | Semantics                                                                                                                           |
| ------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `image` | `String?` | SVG file of a weapon (without extension). For example `"default"` → `assets/svg/ammo/default.svg`. `null` — fallback and `default`. |

## Built-in Collection — reticleImage

In `assets/json/collection.json`, each sight has `reticleImage` and `calibratedMagnification` fields:

```json
{
  "name": "Nightforce ATACR 2.5-10x32",
  "reticleImage": "IHR",
  "calibratedMagnification": 10,
  "focalPlaneValue": "sfp",
  ...
}
```

Examples of identifiers: `"IHR"`, `"DDR-2"`, `"NP-RF1"`, `"LV.5"`, `"MIL-XT"`.

Each ID corresponds to an SVG file from `assets/svg/reticles/<id>.svg`.

## SVG Color Roles

Instead of hard-coded colors, SVG files use Material 3 ColorScheme semantic roles as strings (e.g. `fill="outlineVariant"`). Flutter replaces them with `#rrggbb` when rendered via `resolveSvgColorRoles()` (`lib/core/utils/svg_color_utils.dart`).

Supported roles:

| Role                                                                     | Typical use                           |
| ------------------------------------------------------------------------ | ------------------------------------- |
| `primary`, `onPrimary`, `primaryContainer`, `onPrimaryContainer`         | Accent elements                       |
| `secondary`, `onSecondary`, `secondaryContainer`, `onSecondaryContainer` | Additional elements                   |
| `tertiary`, `onTertiary`, `tertiaryContainer`, `onTertiaryContainer`     | —                                     |
| `error`, `onError`                                                       | —                                     |
| `surface`, `onSurface`, `onSurfaceVariant`                               | Main content                          |
| `outline`, `outlineVariant`                                              | Decorative elements, background icons |
| `onBackground`                                                           | Alias for `onSurface`                 |

Example in SVG: `fill="outlineVariant"` → `fill="#c8c6ca"` (depends on the theme).

## packages/reticle_gen

Arrangement: `packages/reticle_gen/`

**Current status: standalone CLI tool, partially implemented.**

Uses `dart:io` to write SVG to file - cannot be used directly as a Flutter widget.

Launch via `dart run packages/reticle_gen/lib/<file>.dart <output_path>`.

### Architecture

```
MilReticleSVGCanvas   — canvas in mils; viewBox centered (-w/2, -h/2, w, h);
                        pixel size = milWidth×factor × milHeight×factor;
                        data-mil-width / data-mil-height / data-factor in the root <svg>

SVGDrawerInterface    — draw(MilReticleSVGCanvas canvas)

Reticle drawers (lib/):
    ├── default.dart      → DefaultReticleDrawer
    ├── ddr_2.dart        → Ddr2Drawer
    ├── mil_c_f1.dart     → MilCF1Drawer
    ├── mil_r_f1.dart     → MilRF1Drawer
    ├── mil_xt.dart       → MilXtDrawer
    ├── moar.dart         → MoarDrawer
    └── moar_t.dart       → MoarTDrawer

Target drawers (lib/):
    └── target_default.dart → TargetPointDrawer   — dot 0.5×0.5 MIL, color orange

```

### MilReticleSVGCanvas — coordinate system

```dart
MilReticleSVGCanvas(milWidth: 30, milHeight: 30, factor: 100)
// → SVG 3000×3000 px, viewBox (-15, -15, 30, 30) — coordinates in mils
// Coordinates in drawer — in mils: line(-10, 0, 10, 0, ...) = 10 MIL left/right
// unitScale ≠ 1.0 → <g transform="scale(...)"> so drawer can work in MOA etc.
```

### Canvas Methods

| Method                               | Description                                 |
| ------------------------------------ | ------------------------------------------- |
| `line(x1,y1,x2,y2,stroke,sw)`        | Line                                        |
| `rect(x,y,w,h,fill,...)`             | Rectangle                                   |
| `circle(cx,cy,r,fill,...)`           | Circle                                      |
| `dot(cx,cy,r,fill,...)`              | Alias ​​for `circle` (with id-hint `'dot'`) |
| `path(d,fill,...)`                   | SVG path                                    |
| `text(s,x,y,fill,...)`               | Text                                        |
| `fill(color)`                        | Fill entire canvas                          |
| `clip(shape:,draw:)`                 | `<clipPath>` + `<g clip-path>`              |
| `cross(cx,cy,size,stroke,sw)`        | Cross (via PathBuilder)                     |
| `hLine/vLine`                        | Horizontal/Vertical Line                    |
| `hRuler/vRuler`                      | Dashed Lines                                |
| `dashLine/hDashLine/vDashLine`       | Dashed Line                                 |
| `dotLine/hDotLine/vDotLine`          | Dotted Line                                 |
| `dotGrid`                            | Dot Grid                                    |
| `repeat(x1,y1,x2,y2,xStep,yStep,fn)` | Repeat at Equal Intervals                   |
| `batchLines(stroke,sw,fn)`           | Batch via PathBuilder                       |
| `drawAdjustment(x,y)`                | Lines + dot adjustment (legacy, red)        |

### Custom SVG attributes (self-describing)

`MilReticleSVGCanvas.generate()` adds to the root `<svg>`:

| Attribute         | Value                                  |
| ----------------- | -------------------------------------- |
| `data-mil-width`  | Canvas width in mils (default `30.0`)  |
| `data-mil-height` | Canvas height in mils (default `30.0`) |
| `data-factor`     | Scale factor (default `100`)           |
| `shape-rendering` | `crispEdges`                           |

Flutter reads these attributes via `_parseSvgMeta()` to correctly calculate the correction coordinates without hardcoding.

## Current UI state

### HomeReticlePage (`lib/features/home/widgets/home_reticle_page.dart`)

Page 1 (of three) HomeScreen. Consists of two parts:

**`_ReticleView`** — takes 2/3 width, `AspectRatio(1)`:
- Loads SVG via `reticleSvgProvider` (Riverpod, with caching)
- **Runtime color substitution:** `resolveSvgColorRoles()` replaces roles with `#rrggbb` from current `ColorScheme`
- **Injection of correction:** inserts 2 lines + circle before `</svg>` in SVG coordinate system
- **OUT OF RANGE:** if correction goes outside canvas — displays text instead of indicator
- Renders via `flutter_svg`: `SvgPicture.string(fit: BoxFit.contain)`
- Correction color: `orangeAccent` (dark) / `deepOrangeAccent` (light)

**`_AdjPanel`** — takes up 1/3 of the width:
- Shows Drop and Windage from `homeVmProvider` (`AdjustmentData`)
- Format (`AdjustmentDisplayFormat`): arrows / signs / letters — from settings
- If `elevation.isEmpty` — shows `'Enable units...'`

### Weapon / Ammo Collection Tiles

`CollectionWeaponTileBody` and `CollectionAmmoTileBody` display SVG images as the card background via `WeaponSvgView` / `AmmoSvgView`. Fallback is `default.svg` of the corresponding category.

## Reticle Fullscreen Screen

Opens from Home Page 1 (HomeReticlePage) — tap on reticle.
Full-screen view of the reticle with corrections.

**Formula (shared with `DistanceConvertorScreen`):**
```
angularSizeMil = physicalSizeMm / (distanceM * 1000)
```
`ReticleViewScreen` і `HomeScreen`: `ballisticAdj + manualAdj` → `offsetMil`.  
`DistanceConvertorScreen`: `offset = (0, 0)`, `targetSizeMil` з angular-input.

> **Implementation dependency:** `ReticleView` refactor (Phase 3) must be done **by** `DistanceConvertorScreen`.

### ReticleView — dumb widget API

`ReticleView` knows nothing about corrections and calculations. Caller passes already calculated values:

```dart
ReticleView({
  required String? reticleImageId,  // → reticleSvgProvider(id)
  required String? targetImageId,   // → targetSvgProvider(id); null → 'point'
  required double targetSizeMil, // target size in mils
  required double offsetXMil, // X offset (+ = right)
  required double offsetYMil, // Y offset (+ = down in SVG)
})
```

Internal logic:
1. Loads reticle SVG — injects two lines (from center to offset) before `</svg>`
2. Overlays target SVG via `Stack`: positioned at `(offsetX, offsetY)` MIL, size `targetSizeMil`
3. Pixel position = `center ± (offsetMil / milWidth) * widgetSize` (via `data-mil-width` from SVG-meta)

Who transmits what:
| Caller                    | offsetX/Y            | targetSizeMil                                        |
| ------------------------- | -------------------- | ---------------------------------------------------- |
| `HomeReticlePage`         | ballistic correction | fixed or with `ReticleSettings`                      |
| `ReticleViewScreen`       | ballistic + drums    | with `ReticleSettings` + `conditions.targetDistance` |
| `DistanceConvertorScreen` | `(0, 0)`             | with angular-input converter                         |
