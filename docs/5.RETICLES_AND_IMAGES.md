## Entity Fields

### Sight

| Поле | Тип | Семантика |
|---|---|---|
| `reticleImage` | `String?` | Ідентифікатор типу сітки (наприклад `"IHR"`, `"DDR-2"`, `"MIL-XT"`). Береться з `collection.json`, зберігається в OB. Формат — рядок-ідентифікатор, не шлях і не base64. |

```json
{
  "name": "Nightforce ATACR 2.5-10x32",
  "reticleImage": "IHR",
```

## SVG Color Roles

Замість хардкодованих кольорів SVG-файли використовують семантичні ролі Material 3 ColorScheme як рядки (наприклад `fill="outlineVariant"`). Flutter замінює їх на `#rrggbb` при відображенні через `resolveSvgColorRoles()` (`lib/core/utils/svg_color_utils.dart`).

Підтримувані ролі:

| Роль | Типове використання |
|---|---|
| `primary`, `onPrimary`, `primaryContainer`, `onPrimaryContainer` | Акцентні елементи |
| `secondary`, `onSecondary`, `secondaryContainer`, `onSecondaryContainer` | Додаткові елементи |
| `tertiary`, `onTertiary`, `tertiaryContainer`, `onTertiaryContainer` | — |
| `error`, `onError` | — |
| `surface`, `onSurface`, `onSurfaceVariant` | Основний контент |
| `outline`, `outlineVariant` | Декоративні елементи, фонові іконки |
| `onBackground` | Аліас для `onSurface` |

Приклад у SVG: `fill="outlineVariant"` → `fill="#c8c6ca"` (залежить від теми).

## packages/reticle_gen

Розташування: `packages/reticle_gen/`

**Поточний статус: standalone CLI-інструмент, частково реалізований.**

Використовує `dart:io` для запису SVG у файл — не може безпосередньо використовуватись як Flutter widget.

Запуск через `dart run packages/reticle_gen/lib/<file>.dart <output_path>`.

### Архітектура

```
MilReticleSVGCanvas   — canvas у мілах; viewBox центрований (-w/2, -h/2, w, h);
                        pixel size = milWidth×factor × milHeight×factor;
                        data-mil-width / data-mil-height / data-factor у кореневому <svg>

SVGDrawerInterface    — draw(MilReticleSVGCanvas canvas)

Реticle drawers (lib/):
    ├── default.dart      → DefaultReticleDrawer   — простий хрест
    ├── ddr_2.dart        → Ddr2Drawer
    ├── mil_c_f1.dart     → MilCF1Drawer
    ├── mil_r_f1.dart     → MilRF1Drawer
    ├── mil_xt.dart       → MilXtDrawer
    ├── moar.dart         → MoarDrawer
    └── moar_t.dart       → MoarTDrawer

Target drawers (lib/):
    └── target_default.dart → TargetPointDrawer   — крапка 0.5×0.5 MIL, колір orange

```

### MilReticleSVGCanvas — система координат

```dart
MilReticleSVGCanvas(milWidth: 30, milHeight: 30, factor: 100)
// → SVG 3000×3000 px, viewBox (-15, -15, 30, 30) — координати в мілах
// Координати у drawer — в мілах: line(-10, 0, 10, 0, ...) = 10 MIL вліво/вправо
// unitScale ≠ 1.0 → <g transform="scale(...)"> щоб drawer міг працювати в MOA тощо
```

### Методи canvas

| Метод | Опис |
|-------|------|
| `line(x1,y1,x2,y2,stroke,sw)` | Лінія |
| `rect(x,y,w,h,fill,...)` | Прямокутник |
| `circle(cx,cy,r,fill,...)` | Коло |
| `dot(cx,cy,r,fill,...)` | Alias для `circle` (з id-hint `'dot'`) |
| `path(d,fill,...)` | SVG path |
| `text(s,x,y,fill,...)` | Текст |
| `fill(color)` | Заповнення всього canvas |
| `clip(shape:,draw:)` | `<clipPath>` + `<g clip-path>` |
| `cross(cx,cy,size,stroke,sw)` | Хрест (via PathBuilder) |
| `hLine/vLine` | Горизонтальна/вертикальна лінія |
| `hRuler/vRuler` | Риски через рівні інтервали |
| `dashLine/hDashLine/vDashLine` | Штрихова лінія |
| `dotLine/hDotLine/vDotLine` | Лінія з крапок |
| `dotGrid` | Сітка крапок |
| `repeat(x1,y1,x2,y2,xStep,yStep,fn)` | Повтор через рівні інтервали |
| `batchLines(stroke,sw,fn)` | Batch через PathBuilder |
| `drawAdjustment(x,y)` | Лінії + крапка поправки (legacy, red) |

### Кастомні SVG-атрибути (self-describing)

`MilReticleSVGCanvas.generate()` додає до кореневого `<svg>`:

| Атрибут | Значення |
|---------|---------|
| `data-mil-width` | Ширина канвасу в мілах (default `30.0`) |
| `data-mil-height` | Висота канвасу в мілах (default `30.0`) |
| `data-factor` | Масштабний фактор (default `100`) |
| `shape-rendering` | `crispEdges` |

Flutter зчитує ці атрибути через `_parseSvgMeta()` для правильного обчислення координат поправки без хардкоду.

### MilReticleDrawer — перший варіант (тестовий)

- Зовнішнє коло r=15 MIL — обідок + clip-path
- Горизонтальна вісь: лінія −10..+10, риски кожний 1 MIL, числові підписи над рискою (парні)
- Вертикальна вісь: лінія −10..+14, риски кожний 1 MIL, числові підписи зліва (парні)
- Весь вміст обрізається по колу; обідок малюється поверх

## Поточний стан UI

### HomeReticlePage (`lib/features/home/widgets/home_reticle_page.dart`)

Page 1 (з трьох) HomeScreen. Складається з двох частин:

**`_ReticleView`** — займає 2/3 ширини, `AspectRatio(1)`:
- Завантажує SVG через `reticleSvgProvider` (Riverpod, з кешуванням)
- **Runtime підміна кольорів:** `resolveSvgColorRoles()` замінює ролі на `#rrggbb` з поточного `ColorScheme`
- **Ін'єкція поправки:** вставляє 2 лінії + коло перед `</svg>` у системі координат SVG
- **OUT OF RANGE:** якщо поправка виходить за межі canvas — виводить текст замість індикатора
- Рендерить через `flutter_svg`: `SvgPicture.string(fit: BoxFit.contain)`
- Колір поправки: `orangeAccent` (dark) / `deepOrangeAccent` (light)

**`_AdjPanel`** — займає 1/3 ширини:
- Показує Drop і Windage з `homeVmProvider` (`AdjustmentData`)
- Формат (`AdjustmentDisplayFormat`): arrows / signs / letters — з налаштувань
- Якщо `elevation.isEmpty` — показує `'Enable units...'`

## Reticle Fullscreen Screen

Opens from Home Page 1 (HomeReticlePage) — tap on reticle.
Full-screen view of the reticle with corrections.

**Formula (shared with `DistanceConvertorScreen`):**
```
angularSizeMil = physicalSizeMm / (distanceM * 1000)
```
`ReticleViewScreen` і `HomeScreen`: `ballisticAdj + manualAdj` → `offsetMil`.  
`DistanceConvertorScreen`: `offset = (0, 0)`, `targetSizeMil` з angular-input.

> **Implementation dependency:** `ReticleView` рефактор (Phase 3) має бути зроблений **до** `DistanceConvertorScreen`.

### ReticleView — dumb widget API

`ReticleView` нічого не знає про поправки і розрахунки. Caller передає вже обчислені значення:

```dart
ReticleView({
  required String? reticleImageId,  // → reticleSvgProvider(id)
  required String? targetImageId,   // → targetSvgProvider(id); null → 'point'
  required double  targetSizeMil,   // розмір таргету в мілах
  required double  offsetXMil,      // зміщення по X (+ = вправо)
  required double  offsetYMil,      // зміщення по Y (+ = вниз у SVG)
})
```

Внутрішня логіка:
1. Завантажує reticle SVG — інжектує дві лінії (від центру до offset) перед `</svg>`
2. Накладає target SVG через `Stack`: позиціонується у `(offsetX, offsetY)` MIL, розмір `targetSizeMil`
3. Pixel-позиція = `center ± (offsetMil / milWidth) * widgetSize` (через `data-mil-width` з SVG-мета)

Хто передає що:
| Caller | offsetX/Y | targetSizeMil |
|--------|-----------|----------------|
| `HomeReticlePage` | балістична поправка | фіксований або з `ReticleSettings` |
| `ReticleViewScreen` | балістична + барабани | з `ReticleSettings` + `conditions.targetDistance` |
| `DistanceConvertorScreen` | `(0, 0)` | з angular-input конвертора |
