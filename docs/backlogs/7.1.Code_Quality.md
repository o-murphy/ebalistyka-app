## ~~1. Converter Screens — Repeated Structure~~ ✅ DONE

**Problem**: 8 converter screens follow an identical structure but each re-implement the same boilerplate: `_buildInfoTile()`, layout (input picker → divider → info tiles), and provider wiring.

**Applied to**: 5/8 simple screens (length, weight, pressure, temperature, torque).  
**Skipped**: velocity, angular, target_distance — unique layouts, kept as-is.

**What was done**:
- [x] Created `SimpleConvertorScreen` — generic `StatelessWidget` (input picker → sections → info tiles)
- [x] Eliminated `_buildInfoTile()` from all 5 screens — replaced with `InfoListTile` iteration in the generic screen
- [x] Each screen reduced to ~25 lines (thin `ConsumerWidget` wrapper)
- **Actual reduction**: ~270 LOC removed from 5 screens

## ~~4. Table Editor Screens — Missing Abstraction~~ ✅ DONE

**Problem**: `MultiBcEditorScreen` and `PowderSensTableEditorScreen` are nearly identical: list of `TextEditingController`s, identical dispose logic, row-by-row input building, save/discard pattern.

**What was done**:

- [x] `multi_bc_editor_screen.dart`: removed internal `_TwoColumnTableEditorScreen` + all sub-widgets; `MultiBcEditorScreen` and `CustomDragTableEditorScreen` are thin wrappers; 400 → 110 lines
- [x] `powder_sens_table_editor_screen.dart`: replaced `_PowderSensEditorScreen` with `TwoColumnTableEditorScreen`; converted to `ConsumerStatefulWidget` for live preview state; 450 → 195 lines
- **Actual reduction**: ~545 LOC removed

## ~~6. Validated TextField — Repeated Inline Pattern~~ ✅ SUPERSEDED

**Problem**: Every wizard screen hand-builds the same padded, validated `TextField` inline. The pattern `_ctrl.text.trim().isEmpty ? 'Required' : null` appears 20+ times across three files.

**What happened**: The validated required-name pattern was eliminated by task #4 (`WizardNameField`). After that change, the remaining inline `TextField`s in wizard screens are optional fields only (vendor, caliber name) — no validator, padding already correct (`EdgeInsets.fromLTRB(16, 8, 16, 8)`). There are ~3 occurrences across two files, not worth a separate widget.
