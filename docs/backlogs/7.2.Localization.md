## ~~3. `Unit.symbol` / `Unit.label` and `UnitFormatter`~~ ✅ DONE

### What was done

- Created `lib/core/extensions/unit_label_extensions.dart` with two extension methods on `Unit`:
  - `localizedLabel(AppLocalizations l10n)` — full localized name (e.g. "Метри", "Гран")
  - `localizedSymbol(AppLocalizations l10n)` — localized symbol (e.g. "м", "гран", "км/год", "Мах", "МРАД")
- Added 34 × `unitXxxSym` ARB keys covering all picker-relevant units; Ukrainian symbols match UA conventions (`"м/с"`, `"гПа"`, `"мм рт. ст."`, `"км/год"`, `"Мах"`, `"МРАД"`, etc.)
- `unit_picker_button.dart` + `adjustment_input_with_clicks.dart` — both pickers now call `unit.localizedLabel(l10n)` / `unit.localizedSymbol(l10n)`
- `UnitFormatterImpl` — `AppLocalizations _l10n` injected via constructor; `_fmt()` uses `unit.localizedSymbol(_l10n)`; `mach()` uses `_l10n.unitMachSym`; `powderSensitivity()` uses `_l10n.powderSensUnit`; all `xyzSymbol` getters use `localizedSymbol`
- `formatter_provider.dart` — passes `ref.watch(appLocalizationsProvider)` to `UnitFormatterImpl`
- `unit_formatter_test.dart` — updated to use `AppLocalizationsEn()` as second constructor arg; mach test updated (`'M'` → `'Mach'`); all 46 tests pass
